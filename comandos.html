Comandos, variables y funciones

Los comandos nos permiten interactuar con el sistema a través de órdenes enviadas desde una terminal que se ejecuta en el aplicativo 'Gestion de Mosaicos'. Para comenzar, podemos acceder a la sección "Estado de robots", donde podremos consultar el estado actual del sistema. Además, desde esta zona es posible enviar comandos directamente al robot seleccionado para interactuar con él de manera más específica.

Tabla de comandos

# Comando Función Cuando usarlo
1 print prgid[X] Visualiza el numero del mosaico programado en la linea interna del robot 'x'. Reemplazar x por la línea interna a consultar Los mosaicos son por defecto la misma línea. Solo cambia el lado del robot, adicionando 1000 para los lados derechos del sistema. Por ejemplo si la línea 3 es el lado derecho el mosaico de la línea seria el 1003, pero si la línea es el lado izquierdo el valor de la línea seria el 3
2 print prgdone[X] Visualiza el estado de la línea interna del robot 'x'. Reemplazar x por la línea interna a consultar. Si El resultado es: 1 Linea Bloqueada y 0 Linea Desbloqueada Este comando se aplica cuando el Estado de la Linea se encuentra en "Wait Robot Reset" y no cambia de estado
3 prgdone[X]=0 Desbloquea la línea interna del robot seleccionado Este comando se aplica si al consultar con print prgdone[X] el resultado es 1. Al aplicar este comando libera la entrada y el sistema resetea los flags internos de la entrada para iniciar de nuevo la operación
4 print timehoralan[idmosaico] Visualiza el valor del temporizador de empuje cuando recoge el producto identificado con el id mosaico Este comando se aplica cuando queremos conocer el tiempo de empuje que aplica el robot al momento de recoger el producto en la zona de recepción
5 print timehoratras[idmosaico] Visualiza el valor del temporizador de empuje cuando deja el producto identificado con el id mosaico Este comando se aplica cuando queremos conocer el tiempo de empuje que aplica el robot al momento de dejar el producto sobre el pallet
6 print speeddejada Visualiza la velocidad del movimiento de dejada del papel cuando lo deja sobre la estiba o pallet Este comando es util para conocer la velocidad con la que el robot realiza este movimiento especifico de dejar el papel kraft. Aunque es una velocidad estandar, para este movimiento es de 50
7 speeddejada=velocidad Modifica la velocidad del movimiento de dejada del papel cuando lo deja sobre la estiba o pallet. reemplazando velocidad por la velocidad deseada entre 0 a 100 Este comando se aplica para modificar la velocidad con la que el robot realiza este movimiento especifico de dejar el papel kraft La velocidad estandar para este movimiento es de 50
8 pcexec autostart.pc Reinicia la programacion del robot. No borra la programacion, al contrario conrinua la produccion donde quedo. Este comando se aplica cuando una celda no inicia después de ser detenida. En este caso, la baliza se mantiene en rojo estático o se apaga por completo. Este problema puede ocurrir cuando el robot se inicia, se detiene y luego se intenta iniciar repetidamente. También puede suceder si se intenta iniciar la celda después de haber manipulado manualmente el robot, sin corregir la activación del modo 'repeat' desde el mando o controlador
9 print targetprogram[X] Visualiza el estado de la variable targetprogram de la linea interna del robot 'x'. Reemplazar x por la línea interna a consultar. Esta variable es la que se encarga de llevar el control de la construcción del pallet. Este comando se aplica cuando deseamos conocer el estado de la línea de algun robot. Esto nos es util para indentificar en que estado esta la línea y como podemos proceder. Las respuestas pueden ser:
  • 0) Estado inicial. No se esta llevando a cabo ninguna acción
  • 1) Orden de recoger estiba e iniciar mosaico
  • 2) En construcción de mosaico
  • 3) Estado de visualización para finalización desde el wcs para un mosaico.
  • 4) Modo de error. No se obtienen los valores del mosaico programado correctamente
  • 5) Finalización de pallet cuando ha terminado
  • 6) Estado de error. No hay papel kraft
  • 7) Estado de error. No hay papel kraft
  • 8) Estado de error. Papel kraft
10 targetprogram[X]= 0 Este comando libera la entrada 'x' interna del sistema. Al igualar la variable a 0 le decimos al sistema que la entrada se encuentra disponible Este comando se aplica cuando el Estado de la Linea dice "Wait Robot end" o "Send program". Es importante aplicar este comando unicamente en estos casos, ya que al modificar la variable de control del pallet podemos afectar la secuencia del sistema
11 print hightvel Visualiza la velocidad con la que el robot realiza el movimiento para posicionarse a recoger la estiba o el papel Este comando es util para conocer la velocidad con la que el robot realiza este movimiento especifico. Aunque es una velocidad estandar, para este movimiento es de 100 Reducir esta velocidad puede afectar la eficiencia del sistema
12 print medumvel Visualiza la velocidad con la que el robot realiza el movimiento de trasladación de la estiba y papel Este comando es util para conocer la velocidad con la que el robot realiza este movimiento especifico. Aunque es una velocidad estandar, para este movimiento es de 70. Aumentar esta velocidad puede ser riesgoso por la trasladación de la estiba de un punto a otro. No aumentar
13 print slowvel Visualiza la velocidad con la que el robot realiza el movimiento de recoger la estiba y de confirmar el papel Este comando es util para conocer la velocidad con la que el robot realiza este movimiento especifico. Aunque es una velocidad estandar, para este movimiento es de 55. Aumentar esta velocidad puede ser riesgoso por el posicionamiento del robot al momento de recoger la estiba
14 print speedbox Visualiza la velocidad con la que el robot realiza el movimiento de trasladar el producto de la zona de recepción a la dejada sobre la estiba Este comando es util para conocer la velocidad con la que el robot realiza este movimiento especifico. Aunque es una velocidad estandar, para este movimiento es de 100. Con algunas excepciones como los encartonados, donde por seguridad es necesario reducir esta velocidad
15 speedbox = X Modifica la velocidad con la que el robot realiza el movimiento de trasladar el producto de la zona de recepción a la dejada sobre la estiba. Reemplazando X por nuestra velocidad deseada. Este comando es util para reducir la velocidad de este movimiento cuando trabajamos encartonado o presentamos alguna novedad con el agarre del producto o la conformación de fardos
16 print actprio Muestra el estado de la prioridad de llenado de líneas en el sistema. Si el valor es 1, la prioridad está activada, y el robot dará preferencia a la entrada más llena de producto, ignorando las demás. Si el valor es 0, la prioridad está desactivada.
17 actprio = X Permite activar o desactivar la prioridad del sistema.
  • 'actprio=1' activa la prioridad, priorizando las entradas más saturadas del sistema y dejando de lado las demás.
  • 'actprio=0' desactiva la prioridad, permitiendo que el robot recoja de todas las entradas sin preferencia.
 Este comando se utiliza cuando se desea priorizar la recogida de producto en una línea. Es importante tener en cuenta que el sistema seleccionará esta línea a medida que se vayan llenando las entradas. El robot no ignorará por completo las demás, sino que recogerá en la entrada priorizada hasta completar su ciclo. Una vez finalizado, el robot volverá a recoger en las demás entradas. Si alguna otra entrada se llena, el sistema priorizará automáticamente la nueva entrada llena.
18 linepriority = X Permite modificar la entrada que el robot prioriza. Por ejemplo, si se desea forzar al robot a recoger únicamente de una entrada específica, se debe reemplazar "X" por el identificador de la entrada interna del robot deseada. Esto obligará al robot a recoger exclusivamente de esa entrada. Es importante señalar que para utilizar este comando, la prioridad del sistema debe estar activa. Es importante tener en cuenta que, al forzar la recogida en una sola entrada, el robot ignorará las demás entradas hasta que se desactive la prioridad del sistema.
